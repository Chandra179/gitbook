#### Nice to have

| Area                                        | Why Useful                                                    |
| ------------------------------------------- | ------------------------------------------------------------- |
| **Probes (liveness/readiness/startup)**     | Avoid downtime; debug Pod issues                              |
| **Resource limits & requests (CPU/memory)** | Prevent OOMKilled & throttling                                |
| **HPA (auto-scaling)**                      | Handle traffic spikes automatically                           |
| **Rolling updates / canary deployments**    | Zero-downtime updates                                         |
| **Secrets management**                      | Realistic production secret handling (Vault, ExternalSecrets) |
| **Logs & metrics**                          | Easy troubleshooting, observability                           |
| **Ingress basics**                          | Routing traffic to multiple services                          |
