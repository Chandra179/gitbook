## What's Still Accurate

- **ISR (In-Sync Replicas)** — concept and `replica.lag.time.max.ms` (30s), `min.insync.replicas` still correct. However, Kafka 4.0 also introduced ELR (Eligible Leader Replicas, KIP-966) which adds a new replica set beyond ISR.
- **Controller Broker** — still accurate; now exclusively uses KRaft quorum.
- **Topic / Partition distribution** — still accurate.
- **Producer key-based hash, round-robin, custom partitioner** — still accurate. (Note: `DefaultPartitioner` and `UniformStickyPartitioner` classes were removed in 4.0; the concepts remain.)
- **acks=0/1/all** — still accurate.
- **Idempotent Producer** — still accurate.
- **Message ordering within a partition** — still accurate.
- **Consumer group logic** — still accurate; group coordinator formula (hash mod N) unchanged.
- **Consumer offsets / commits / at-least-once vs at-most-once** — still accurate.
- **Consumer lag** definition — still accurate.

## Potential Updates Needed

1. **ZooKeeper removal is now complete — the page presents it as "future"** — The page says ZooKeeper mode is "Deprecated as of KIP-833, targeted for removal in Kafka 4.0." Kafka 4.0 was released March 18, 2025, and ZooKeeper mode is fully removed. KRaft is now the **only** mode. The page should be updated to remove ZooKeeper as a current option and present KRaft as the sole metadata management system. Source: [Kafka 4.0 Announcement](https://kafka.apache.org/blog/2025/03/18/apache-kafka-4.0.0-release-announcement/)

2. **Missing: Kafka 4.x versions — the page has no version context** — The page never mentions Kafka version numbers. Current latest stable is **4.3.0** (released May 22, 2026). Supported releases as of June 2026: 4.3.0, 4.2.1, 4.1.2, 4.0.2, 3.9.2. A version note would help readers orient themselves. Source: [Kafka Downloads](https://kafka.apache.org/community/downloads/)

3. **Missing: New Consumer Group Protocol (KIP-848) — GA in Kafka 4.0** — The page describes "cooperative sticky rebalancing (modern, Kafka ≥ 2.4)" but Kafka 4.0 introduced a entirely new consumer group coordinator with a next-generation rebalance protocol (KIP-848). This is now GA and auto-enabled on upgrade to 4.0. It dramatically improves rebalance performance, especially at scale. The page should add a section on this. Source: [KIP-848](https://cwiki.apache.org/confluence/display/KAFKA/KIP-848+The+Next+Generation+of+the+Consumer+Rebalance+Protocol)

4. **Missing: Queues for Kafka / Share Groups (KIP-932) — production-ready in 4.2+** — Kafka now supports share groups — a new cooperative consumption model where multiple consumers can concurrently process records from the same partitions with individual acknowledgment and delivery counting. This enables queue-like use cases without strict partition-to-consumer assignment. This is a major new feature category that deserves its own section. Source: [KIP-932](https://cwiki.apache.org/confluence/display/KAFKA/KIP-932+Queues+for+Kafka)

5. **Missing: Eligible Leader Replicas (KIP-966) — new in Kafka 4.0** — Beyond ISR, Kafka 4.0 introduces ELR — replicas not in ISR but safe to elect as leader without data loss. This enhances availability during broker failures. The ELR section should follow the existing ISR section. Source: [KIP-966](https://cwiki.apache.org/confluence/display/KAFKA/KIP-966+Eligible+Leader+Replicas)

6. **Missing: Transactions Server Side Defense (KIP-890) — new in Kafka 4.0** — Production epoch is now bumped on every transaction, preventing duplicate writes during transaction replays. Relevant if using exactly-once semantics. Source: [KIP-890](https://cwiki.apache.org/confluence/display/KAFKA/KIP-890+Transactions+Server+Side+Defense)

7. **Missing: Java requirements changed in 4.0** — Kafka 4.0 requires Java 17 for brokers/connect/tools and Java 11 for clients/streams. The page doesn't mention Java requirements at all. Worth adding a note.

8. **Missing: Tiered Storage** — Kafka now supports tiered storage (offloading old segments to cheaper storage like S3/GCS). Configs like `remote.log.manager.follower.thread.pool.size` and `follower.fetch.last.tiered.offset.enable` (KIP-1023) are relevant. The page covers log segments but not tiered offloading. Source: [KIP-1023](https://cwiki.apache.org/confluence/display/KAFKA/KIP-1023+Allow+followers+to+fetch+from+the+last+tiered+offset)

9. **MirrorMaker (MM1) removed in 4.0** — The page doesn't mention mirroring/replication, but if added, it should note that MM1 is removed; only MM2 (Connect-based) is available.

10. **Deprecated/Removed items in 4.0 worth noting** — The page references the old `log.message.format.version` config implicitly by not mentioning it, but if someone is upgrading from a pre-4.0 cluster, they need to know this config was removed. Also `offsets.commit.required.acks` removed.

11. **Kafka 4.3 specific additions (May 2026)** — New `share.delivery.count.limit` and `share.partition.max.record.locks` configs for share groups; `group.coordinator.background.threads` config; `remote.log.metadata.topic.min.isr` config (default 2); kafka-streams-scala library deprecated. These are very recent and less critical but worth noting for completeness. Source: [Kafka 4.3 Upgrade Notes](https://kafka.apache.org/43/getting-started/upgrade/)

12. **`kafka-streams-scala` deprecated as of 4.3** — Will be removed in Kafka 5.0. Migrate to pure Java API. Source: [KIP-1237](https://cwiki.apache.org/confluence/display/KAFKA/KIP-1237)
